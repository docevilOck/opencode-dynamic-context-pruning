# 项目架构总览

## 项目简介
- 项目名称：Dynamic Context Pruning / `@tarquinen/opencode-dcp`
- 技术栈：TypeScript、Node.js ESM、OpenCode Plugin SDK
- 一句话定位：OpenCode 的上下文裁剪插件，通过压缩、去重、错误清理和提示词注入，降低会话 token 占用。

## 仓库顶层目录结构

- `index.ts` - 根运行时入口，拼接插件生命周期和工具
- `lib/` - 主源码，覆盖模块 02 到 09
- `scripts/` - 本地诊断、打印和发布校验工具，不随包发布
- `tests/` - 单元测试和回归脚本
- `docs/plan/` - 既有实现计划文档，当前仓库里已有 `26-05-23_dcp_compress_backend_plan.md`
- `docs/architecture/` - 本套导航文档
- `assets/images/` - README 示例截图
- `package.json` - 入口、脚本、发布配置
- `tsup.config.ts` - 打包入口配置
- `tsconfig.json` - TypeScript 编译配置
- `dcp.schema.json` - 配置 schema
- `README.md` / `README.zh-CN.md` - 用户文档

## 读写规则

- 在修改代码之前，先阅读对应模块文档，确认职责边界和目录映射
- 架构变更时，优先同步更新本目录下的模块文档和根 `AGENTS.md`
- 若文档写了“仓库目录映射”，以映射为准，不要靠文件名猜路径
