# 会话状态与持久化

## 概述
这一层保存所有会话级状态：压缩块、消息引用、工具参数缓存、nudge 锚点、统计信息和压缩时间。它是消息处理链路和压缩引擎共享的“状态底座”。

## 仓库目录映射
- 主目录：`lib/state/`
- 核心文件：
  - `lib/state/types.ts` - `SessionState`、`CompressionBlock`、`PruneMessagesState` 等核心结构
  - `lib/state/state.ts` - `checkSession()`、`createSessionState()`、`ensureSessionInitialized()`
  - `lib/state/persistence.ts` - 会话状态序列化/反序列化与统计加载
  - `lib/state/tool-cache.ts` - 工具调用参数和 token 统计缓存
  - `lib/state/utils.ts` - compaction、turn 计数、nudge anchor、summary usage 等辅助
  - `lib/message-ids.ts` - 生成和解析 `mNNNN` / `bN` 这类消息和块引用
  - `lib/token-utils.ts` - token 估算、消息 token 汇总和系统 prompt token 估计
- 相关测试：`tests/message-ids.test.ts`、`tests/token-counting.test.ts`、`tests/token-usage.test.ts`
- 依赖的其他模块：`消息处理链路`、`压缩引擎`、`辅助与工具`

```text
├─ toolParameters

├─ toolIdList

```

## 关键流程
1. `checkSession()` 根据最后一个 user message 判断 session 是否切换。
2. `ensureSessionInitialized()` 在 session 切换时重置状态、加载持久化数据、识别 subagent 会话。
3. `persistence.ts` 把 `prune.messages`、nudge 锚点和统计信息写回磁盘。
4. `tool-cache.ts` 记录工具参数、状态和 token 数，供通知和裁剪文案复用。
5. `message-ids.ts` 给消息和压缩块分配稳定引用，供注入、通知和压缩块定位使用。

## 主要数据结构

### `SessionState`
- 定义位置：`lib/state/types.ts:93`
- 用途：会话级共享状态的总入口。
- 关键字段：
  - `prune` - 压缩块和已裁剪消息的状态
  - `nudges` - 各类 nudge 锚点集合
  - `nudges.lastCompressionMessageCount` - 上次成功压缩时的非忽略消息计数，用于压缩后提醒冷却
  - `stats` - 累计裁剪 token 统计
  - `toolParameters` - 工具输入缓存
  - `messageIds` - 原始 ID 到引用 ID 的映射

### `CompressionBlock`
- 定义位置：`lib/state/types.ts:33`
- 用途：记录一轮压缩的范围、摘要、活动状态和父子块关系。

### `PruneMessagesState`
- 定义位置：`lib/state/types.ts:62`
- 用途：保存消息级压缩索引，驱动 summary 注入和恢复。

### `MessageIdState`
- 定义位置：`lib/state/types.ts:81`
- 用途：维护 `mNNNN` / `bN` 引用的双向映射。

### `ToolParameterEntry`
- 定义位置：`lib/state/types.ts:11`
- 用途：记录工具名、参数、状态、错误和 token 数。
