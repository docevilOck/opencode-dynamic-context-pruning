# 压缩引擎

## 概述
这一层实现真正的裁剪工具：`range` 模式压缩消息区间，`message` 模式压缩单条消息。它会先准备会话、解析范围、拼装保护内容、计算 token、写入压缩块状态，最后发送通知并持久化。

## 关键流程
1. `createCompressRangeTool()` 和 `createCompressMessageTool()` 从 prompt store 取运行时描述，再定义 tool schema。
2. `prepareSession()` 在每次工具调用前统一做权限申请、会话初始化、消息拉取和策略预处理。
3. `range.ts` 解析边界，校验 placeholder，并把旧 block summary 重新嵌入新摘要。
4. `message.ts` 直接按单条消息生成摘要，适合更细粒度的压缩。
5. `state.ts` 把新的压缩块写入 `state.prune.messages`，维护父子块和活动块集合。
6. `finalizeSession()` 负责保存状态和发送压缩通知。

## 主要数据结构

### `ToolContext`
- 定义位置：`lib/compress/types.ts:6`
- 用途：压缩工具执行所需的 client/state/logger/config/prompts 上下文。

### `CompressionStateInput`
- 定义位置：`lib/compress/types.ts:98`
- 用途：把一次压缩写入 `CompressionBlock` 所需的最小输入封装起来。

### `ResolvedRangeCompression`
- 定义位置：`lib/compress/types.ts:66`
- 用途：range 模式下每个压缩区间的解析结果。

### `ResolvedMessageCompression`
- 定义位置：`lib/compress/types.ts:60`
- 用途：message 模式下每条消息压缩计划的解析结果。

### `InjectedSummaryResult`
- 定义位置：`lib/compress/types.ts:86`
- 用途：记录 block placeholder 展开后的摘要和 consumed block。

### `AppliedCompressionResult`
- 定义位置：`lib/compress/types.ts:91`
- 用途：记录压缩后新增的 message/tool 影响面。
