# 消息处理链路

## 概述
这一层负责把原始消息变成可用于压缩和展示的稳定输入。它包含消息形状过滤、最后一条 user message 定位、优先级评估、压缩块同步、消息注入、以及 subagent 结果回填。

## 仓库目录映射
- 主目录：`lib/messages/`
- 核心文件：
  - `lib/messages/shape.ts` - 只保留结构合法的 `WithParts`
  - `lib/messages/query.ts` - 最后一个 user message、压缩标记、受保护 user message 判定
  - `lib/messages/priority.ts` - message mode 下的优先级映射
  - `lib/messages/prune.ts` - 把已裁剪内容替换为占位文本并注入压缩摘要
  - `lib/messages/sync.ts` - 根据当前消息集同步压缩块活动状态
  - `lib/messages/utils.ts` - synthetic message / text part、tool id 列表、hallucination 清理
  - `lib/messages/inject/inject.ts` - nudge 和 message id 注入
  - `lib/messages/inject/subagent-results.ts` - subagent task 输出回填
  - `lib/messages/inject/utils.ts` - 注入辅助与 priority guidance
  - `lib/subagents/subagent-results.ts` - 从 subagent 会话提取最终文本并合并到 task output
  - `lib/message-ids.ts` - 消息和压缩块引用格式化、解析、tag 注入
  - `lib/protected-patterns.ts` - 文件路径和工具名保护匹配
- 相关测试：`tests/message-ids.test.ts`、`tests/message-priority.test.ts`、`tests/message-utils.test.ts`、`tests/message-ids.test.ts`、`tests/hooks-permission.test.ts`
- 依赖的其他模块：`会话状态与持久化`、`配置与权限`、`压缩引擎`

## 关键流程
1. `hooks.ts` 进入 chat transform 后，先用 `filterMessagesInPlace()` 剔除非法形状。
2. `checkSession()` 之后，`syncCompressPermissionState()` 决定是否允许继续。
3. `stripHallucinations()`、`assignMessageRefs()`、`syncCompressionBlocks()` 和 `syncToolCache()` 先把消息整理成可压缩状态。
4. `prune()` 注入压缩摘要并把冗余内容替换成占位文本。
5. `injectExtendedSubAgentResults()` 负责把 subagent 的 task 输出展开回填。
6. `buildPriorityMap()` 和 `injectMessageIds()` 为 message mode 提供优先级和引用标签。

## 主要数据结构

### `CompressionPriorityMap`
- 定义位置：`lib/messages/priority.ts:18`
- 用途：以原始 message ID 为 key，保存引用、token 数和优先级。

### `CompressionPriorityEntry`
- 定义位置：`lib/messages/priority.ts:12`
- 用途：保存单条消息的引用和 priority。

### `LastUserModelContext`
- 定义位置：`lib/messages/inject/utils.ts:27`
- 用途：保存最后一个 user message 的 provider/model 信息，用于 nudge 生成。

### `LastNonIgnoredMessage`
- 定义位置：`lib/messages/inject/utils.ts:32`
- 用途：记录最后一个未被忽略的消息及其索引，供 nudge 锚点计算。

### `CompressionTarget`
- 定义位置：`lib/commands/compression-targets.ts:3`
- 用途：压缩和恢复时的目标分组视图，虽然定义在 commands 目录，但由消息链路驱动。
