# 命令系统

## 概述
这一层实现 `/dcp` 的所有子命令，并由 `hooks.ts` 的 `command.execute.before` 拦截调用。它既负责帮助信息和状态查看，也负责手动压缩、解压和重新压缩。

## 仓库目录映射
- 主目录：`lib/commands/`
- 核心文件：
  - `lib/commands/index.ts` - 子命令聚合入口
  - `lib/commands/context.ts` - 显示当前上下文 token 拆分
  - `lib/commands/help.ts` - 列出可见命令
  - `lib/commands/manual.ts` - 手动模式开关和手动触发
  - `lib/commands/stats.ts` - 显示累计裁剪统计
  - `lib/commands/sweep.ts` - 裁剪某个区间之后的工具输出
  - `lib/commands/decompress.ts` - 恢复压缩块
  - `lib/commands/recompress.ts` - 重新应用用户解压过的压缩块
  - `lib/commands/compression-targets.ts` - 压缩块分组、可恢复目标和目标解析
- 相关入口：`lib/hooks.ts`
- 相关测试：`tests/compression-targets.test.ts`、`tests/decompress.test.ts`、`tests/recompress.test.ts`、`tests/hooks-permission.test.ts`
- 依赖的其他模块：`会话状态与持久化`、`消息处理链路`、`压缩引擎`、`辅助与工具`

## 关键流程
1. `hooks.ts` 在命令执行前拦截 `/dcp`。
2. `handleHelpCommand()` 负责默认降级路径，确保未知子命令能显示说明。
3. `handleManualTriggerCommand()` 会把压缩请求变成一次待执行的手动触发，而不是立即执行工具。
4. `handleDecompressCommand()` 根据块 ID 选择活动压缩块并撤销它。
5. `handleRecompressCommand()` 只允许重新应用用户解压过的块。
6. `handleSweepCommand()` 通过消息索引定位需要裁剪的工具输出。

## 主要数据结构

### `ContextCommandContext`
- 定义位置：`lib/commands/context.ts:52`
- 用途：`/dcp context` 的执行上下文。

### `ManualCommandContext`
- 定义位置：`lib/commands/manual.ts:48`
- 用途：手动模式切换和手动触发的执行上下文。

### `DecompressCommandContext`
- 定义位置：`lib/commands/decompress.ts:15`
- 用途：解压单个压缩块的执行上下文。

### `RecompressCommandContext`
- 定义位置：`lib/commands/recompress.ts:15`
- 用途：重新应用压缩块的执行上下文。

### `SweepCommandContext`
- 定义位置：`lib/commands/sweep.ts:27`
- 用途：裁剪工具输出的执行上下文。

### `CompressionTarget`
- 定义位置：`lib/commands/compression-targets.ts:3`
- 用途：把单个压缩块或同一 run 的 message-mode 块组织成可恢复目标。
