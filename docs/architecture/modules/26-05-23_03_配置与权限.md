# 配置与权限

## 概述
这一层负责两件事：把多层配置合并成最终 `PluginConfig`，以及把宿主的权限约束折叠成最终的 `compress` 可用权限。配置读错或权限被禁用时，后续的压缩工具和命令都应短路。

## 仓库目录映射
- 主目录：`lib/config.ts`
- 核心文件：
  - `lib/config.ts` - 读取全局 / `$OPENCODE_CONFIG_DIR` / 项目 `.opencode` 配置并合并
  - `lib/host-permissions.ts` - 解析宿主权限规则并判断 compress 是否被禁用
  - `lib/compress-permission.ts` - 把配置值和宿主权限合成最终 compress 权限
- 相关配置：`dcp.schema.json`、`package.json`
- 相关测试：`tests/host-permissions.test.ts`、`tests/hooks-permission.test.ts`、`tests/update.test.ts`
- 依赖的其他模块：`启动与宿主集成`、`会话状态与持久化`、`辅助与工具`

## 架构图
```text
global config
      │
      v
$OPENCODE_CONFIG_DIR config
      │
      v
project .opencode/dcp.jsonc
      │
      v
getConfig() -> PluginConfig
      │
      v
hostPermissions + state.compressPermission
      │
      v
compressPermission()
```

## 关键流程
1. `getConfig()` 先拷贝默认配置，再按 global -> configDir -> project 的顺序覆盖。
2. 配置文件里未知 key 或类型错误会被收集并通过 toast 提示，但不会中断加载。
3. `host-permissions.ts` 先判断宿主是否把 `compress` 设成 deny。
4. `compress-permission.ts` 再把宿主权限和插件默认权限合并成最终可用权限。
5. 下游模块只读取 `compressPermission(state, config)`，不直接重新推导权限。
6. `compress.postCompressionNudgeCooldownMessages` 控制成功压缩后的自动提醒与自动压缩执行冷却期，默认 `5`，可显式设为 `0` 关闭冷却。

## 主要数据结构

### `PermissionConfig`
- 定义位置：`lib/host-permissions.ts:5`
- 用途：表示宿主权限配置，支持全局权限和按 tool/pattern 的细粒度规则。

### `HostPermissionSnapshot`
- 定义位置：`lib/host-permissions.ts:7`
- 用途：缓存当前会话的全局权限和 agent 权限。

### `PluginConfig`
- 定义位置：`lib/config.ts`
- 用途：插件最终配置树，供所有运行时模块读取。
