# 启动与宿主集成

## 概述
这一层只负责把插件装起来，不承载裁剪算法本身。`index.ts` 创建配置、状态、提示词和支持模块，然后把一组 hook/工具返回给 OpenCode；`hooks.ts` 再把运行时事件分发到消息处理、压缩、命令和事件统计。

## 仓库目录映射
- 主目录：`index.ts`
- 核心文件：
  - `index.ts` - 插件入口，决定启用状态并组装返回值
  - `lib/hooks.ts` - system/chat/command/text-complete/event 五类 hook 编排
  - `lib/auth.ts` - 在安全模式下给 SDK client 注入 Authorization 头
  - `lib/update.ts` - 启动时检查 npm 最新版本并清理旧包装目录
- 相关配置：`package.json` 的 `dev`、`build`、`prepublishOnly`
- 相关测试：`tests/hooks-permission.test.ts`、`tests/update.test.ts`
- 依赖的其他模块：`配置与权限`、`会话状态与持久化`、`消息处理链路`、`压缩引擎`、`命令系统`、`提示词系统`

## 架构图
```text
OpenCode
  │
  v
index.ts
  ├─ getConfig()
  ├─ createSessionState()
  ├─ PromptStore()
  ├─ configureClientAuth()
  └─ startAutoUpdate()
  │
  v
hooks.ts
  ├─ system prompt transform
  ├─ chat message transform
  ├─ command execute
  ├─ text complete
  └─ event
```

## 关键流程
1. `index.ts` 读取配置并判断是否启用插件。
2. 创建 `Logger`、`SessionState` 和 `PromptStore`。
3. 在 secure mode 下调用 `configureClientAuth()`。
4. 调用 `startAutoUpdate()` 后返回 hook/tool/config 处理器。
5. `hooks.ts` 接收 OpenCode 生命周期事件，把消息/命令/事件分发给下游模块。

## 主要数据结构

### `PluginConfig`
- 定义位置：`lib/config.ts`
- 用途：描述插件的总配置树，包含 `compress`、`commands`、`manualMode`、`strategies` 等子配置。

### `SessionState`
- 定义位置：`lib/state/types.ts:93`
- 用途：承载会话级运行态，供 hook、消息处理和压缩引擎共享。

### `HostPermissionSnapshot`
- 定义位置：`lib/host-permissions.ts:7`
- 用途：缓存宿主全局权限和 agent 级权限，供 `compressPermission()` 决策。
