# 提示词系统

## 概述
这一层负责把内置 prompt、可编辑 prompt、系统扩展和提醒文案拼成运行时可用内容。它既给 system prompt 注入 DCP 行为，也给 compress tool 提供模式说明和 nudge 文案。

## 仓库目录映射
- 主目录：`lib/prompts/`
- 核心文件：
  - `lib/prompts/index.ts` - `PromptStore`、`RuntimePrompts` 和 prompt 读取/覆盖逻辑
  - `lib/prompts/store.ts` - prompt 文件发现、默认文件生成、override 选择和 reload
  - `lib/prompts/system.ts` - system prompt 基础文本
  - `lib/prompts/compress-range.ts` - range 模式压缩说明
  - `lib/prompts/compress-message.ts` - message 模式压缩说明
  - `lib/prompts/context-limit-nudge.ts` - 超过 max 上下文时的提醒
  - `lib/prompts/turn-nudge.ts` - turn 边界提醒
  - `lib/prompts/iteration-nudge.ts` - 长迭代提醒
  - `lib/prompts/extensions/system.ts` - manual / subagent / protected tools 扩展
  - `lib/prompts/extensions/tool.ts` - tool format schema 扩展
  - `lib/prompts/extensions/nudge.ts` - nudge guidance 拼接
- 相关测试：`tests/prompts.test.ts`、`tests/prompts-store.test.ts`
- 依赖的其他模块：`启动与宿主集成`、`压缩引擎`、`消息处理链路`

```text
  ├─ turn-nudge
  └─ iteration-nudge
```

## 关键流程
1. `PromptStore` 先装载内置 prompt，再按 override 优先级覆盖。
2. `customPrompts` 打开时，会自动生成默认文件和说明 README。
3. `renderSystemPrompt()` 把 system prompt、manual extension、subagent extension 和 protected tools extension 拼接成最终系统提示词。
4. `extensions/tool.ts` 给 compress tool schema 说明提供稳定格式，不允许通过 override 单独修改。
5. `extensions/nudge.ts` 把压缩块上下文和 message priority guidance 插入 nudge 文本。

## 主要数据结构

### `PromptKey`
- 定义位置：`lib/prompts/store.ts:13`
- 用途：标识 6 个可编辑 prompt 文件。

### `PromptDefinition`
- 定义位置：`lib/prompts/store.ts:29`
- 用途：描述 prompt 文件名、标签、用途和 runtime 字段映射。

### `PromptPaths`
- 定义位置：`lib/prompts/store.ts:42`
- 用途：记录默认目录、全局 overrides、config-dir overrides 和 project overrides。

### `RuntimePrompts`
- 定义位置：`lib/prompts/store.ts:49`
- 用途：运行时可直接读取的 prompt 集合。

### `PromptStore`
- 定义位置：`lib/prompts/store.ts:324`
- 用途：prompt 的运行时装配器和 override 选择器。
