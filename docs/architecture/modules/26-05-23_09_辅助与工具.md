# 辅助与工具

## 概述
这一层放的是不直接做裁剪决策、但支撑整个插件运行的工具代码：日志、token 统计、通知、认证、自动更新和展示格式化。

## 仓库目录映射
- 主目录：`lib/logger.ts`、`lib/token-utils.ts`、`lib/auth.ts`、`lib/update.ts`、`lib/ui/`
- 核心文件：
  - `lib/logger.ts` - 结构化日志和上下文快照落盘
  - `lib/token-utils.ts` - token 计数、消息/工具 token 汇总、当前上下文 token 估算
  - `lib/ui/notification.ts` - 压缩通知、忽略消息通知和 toast / chat 输出
  - `lib/ui/utils.ts` - token / progress bar / pruned items 文案格式化
  - `lib/auth.ts` - OpenCode SDK 请求认证头注入
  - `lib/update.ts` - npm 最新版本检查和旧安装目录清理
  - `lib/protected-patterns.ts` - 路径和工具名保护匹配
- 相关测试：`tests/update.test.ts`、`tests/token-usage.test.ts`、`tests/host-permissions.test.ts`
- 依赖的其他模块：所有运行时模块

## 架构图
```text
core modules
  │
  ├─ logger.ts -> log / saveContext
  ├─ token-utils.ts -> count / estimate / summarize
  ├─ ui/notification.ts -> toast or chat notification
  ├─ ui/utils.ts -> formatting helpers
  ├─ auth.ts -> request auth header
  └─ update.ts -> auto update check
```

## 关键流程
1. `Logger` 把调试上下文写到 `~/.config/opencode/logs/dcp/`。
2. `token-utils.ts` 统一使用 Anthropic tokenizer，失败时回退近似估算。
3. `sendCompressNotification()` 和 `sendIgnoredMessage()` 负责压缩结果和忽略输出的最终用户可见消息。
4. `configureClientAuth()` 在 secure mode 下给 SDK client 请求加 Authorization。
5. `startAutoUpdate()` 启动后异步检查 npm registry 并在需要时提示重启。

## 主要数据结构

### `Logger`
- 定义位置：`lib/logger.ts:6`
- 用途：按组件名写 INFO / DEBUG / WARN / ERROR 日志和上下文快照。

### `UpdateResult`
- 定义位置：`lib/update.ts:12`
- 用途：自动更新检查结果的联合类型。

### `PruneReason`
- 定义位置：`lib/ui/notification.ts:13`
- 用途：压缩通知里显示的原因标签。

### `NotificationEntry`
- 定义位置：`lib/ui/notification.ts:25`
- 用途：压缩通知的条目摘要。

### `COMPACTED_TOOL_OUTPUT_PLACEHOLDER`
- 定义位置：`lib/token-utils.ts:83`
- 用途：被压缩的工具输出占位文本。
